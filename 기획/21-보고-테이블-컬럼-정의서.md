# 21. 보고 테이블 컬럼 정의서

마이페이지(개인)와 팀 공유 페이지에서 보고 타입별 테이블 컬럼 구성을 정의한다.
팀 공유 페이지는 마이페이지와 동일한 ReportTable 컴포넌트를 재사용하며, 스타일(색상, 레이아웃, 태그 그룹핑, 정렬)도 동일하게 적용한다.

---

## 일간 보고

### 마이페이지 (/reports/daily)

| 컬럼 | 설명 |
|------|------|
| 주간 태그 | detail.tagId 기준 주간 태그명 + 태그별 투입시간/투입률. 태그별 rowSpan 그룹핑 |
| 업무명 | taskTitle (링크 있으면 하이퍼링크) |
| 업무 성과 | 버튼 형태 — 성과 있으면 "성과 수정"(파란), 없으면 "성과 입력"(주황) |
| 투입률 | workHours + % |
| 한 일 | done 텍스트 |
| 액션 버튼 | 업무 상세 보기, 동료 협업 기록, 내일 할 일 (3개 버튼, rowSpan) |

### 팀 공유 (/team/share?tab=daily)

| 컬럼 | 설명 |
|------|------|
| 주간 태그 | 마이페이지와 동일 |
| 업무명 | 마이페이지와 동일 |
| 투입률 | 마이페이지와 동일 |
| 한 일 | 마이페이지와 동일 |
| 내일 할 일 | report.tomorrowPlan 표시 (report 단위 rowSpan) |

> 마이페이지 대비 차이: **업무 성과** 컬럼 제거, **액션 버튼** 제거, **내일 할 일** 컬럼 추가

---

## 주간 보고

### 마이페이지 (/reports/weekly)

| 컬럼 | 설명 |
|------|------|
| 월간 태그 | detail.tagId의 parentTagId 기준 월간 태그명 + 태그별 투입시간/투입률 |
| 업무명 | taskTitle (링크 있으면 하이퍼링크) |
| 업무 성과 | 버튼 형태 — 성과 수정/성과 입력 |
| 투입률 | workHours + % |
| AI 요약 | reportDetail.aiSummary 표시. 셀 우측에 "수정" 버튼 → 인라인 편집 가능 |
| 한 일 | 버튼 형태 — "한 일 보기" 클릭 시 펼침. 주간 업무에서 한 일은 상대적으로 덜 중요하므로 AI 요약을 우선 노출 |

> 한 일보다 AI 요약이 더 중요한 정보로 판단하여 AI 요약을 먼저 배치

### 팀 공유 (/team/share?tab=weekly)

| 컬럼 | 설명 |
|------|------|
| 월간 태그 | 마이페이지와 동일 |
| 업무명 | 마이페이지와 동일 |
| 투입률 | 마이페이지와 동일 |
| 한 일 요약 | reportDetail.aiSummary 활용. 헤더명이 "한 일 요약"으로 변경됨 (사용자가 수정 가능한 검토 데이터) |
| 한 일 | 버튼 형태 — 마이페이지와 동일 |

> 마이페이지 대비 차이: **업무 성과** 컬럼 제거, AI 요약 → **한 일 요약**으로 헤더명 변경 (데이터는 동일하게 aiSummary 사용)

---

## 월간 보고

### 마이페이지 (/reports/monthly)

주간 보고 마이페이지와 동일한 컬럼 구성. 태그만 **월간 태그**로 표시.

### 팀 공유 (/team/share?tab=monthly)

주간 보고 팀 공유와 동일한 컬럼 구성. 태그만 **월간 태그**로 표시.

---

## 공통 규칙

- **태그 그룹핑**: 일간=주간태그, 주간/월간=월간태그 기준 그룹핑
- **정렬**: 태그 그룹 및 태그 내 업무 모두 투입시간(투입률) 내림차순
- **테이블 분리**: 보고 건별 독립 테이블, 간격(space-y-5) 적용
- **날짜 헤더**: 테이블 상단 좌측에 날짜 + 투입시간 표시 (일간: 근무일, 주간/월간: 시작일~종료일)
- **컬럼 헤더 스타일**: bg-gray-200 배경 + 검정 글자
- **팀 공유 스타일**: 마이페이지와 완전히 동일한 ReportTable 컴포넌트 재사용
- **팀 공유 멤버 구분**: 멤버명을 테이블 그룹 상단 헤더로 표시
