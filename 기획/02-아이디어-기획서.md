# 2. 아이디어 기획서

## Track

**Track 2. 업무 개선**

## 아이디어 개요

- 슈퍼맨과 스크럼을 합친 슈크럼은 매일 작성한 스크럼을 AI가 분석해 주간·월간·연말 리뷰를 자동으로 만들어주는 생산성 도구입니다.
- 보고 작성을 위해 쓰는 시간을 줄여 단순 작업 대신 실제 업무와 성과에 더 집중할 수 있게 합니다.
- 흩어진 그룹사의 업무 공유 방식을 하나로 통일해 표준화된 업무 데이터 기반을 만듭니다.

## 개발 목표 및 범위

- 데일리 스크럼 등록 → 주간보고, 월간 공수(GRM), 리뷰 성과 데이터(동료리뷰, 본인리뷰) 자동화.
- 두레이 로그인 연동 제외: DOORAY_API_TOKEN 입력을 로그인으로 간주. 추후 두레이 채택 시 연동.
- 팀: 1인 1팀 원칙 (겸직/TF 추후 개선).
- 최종 결과물: 웹 애플리케이션 배포, 두레이 SESSION 입력 후 내부 API 활용 예정.

## AI 활용 전략

- **개발 과정**: Claude Code agent 개발, Figma Make UI 생성, 배포 자동화, AI-DLC 워크플로우 도입.
- **결과물**: 성과 보고 산출물 AI 자동 생성.

## 기대효과 및 활용 방안

- 사내 적용: 전사.
- 주간/월간/리뷰 작성 리소스 → 0, 일간 기록만 수행(건당 약 5분), 3개사 기준 연간 약 19억원 규모 생산성 확보, 리뷰 시즌 부담 해소.

## 실현 가능성

- FE: React, TailwindCSS, 구현 페이지 약 6개. AI 기반 도구로 단시간 구현.
- BE: Spring Boot, MySQL, MyBatis 등. **아래 표의 예상 소요·비고는 BE 담당 채우기.**
- 상세 기획은 하단 문서(03~08) 참고.
