# 19. 페이지별 기능 정의서

16-기능정의서.md의 31개 기능을 페이지 단위로 재편성. 배치/수동 생성 기능은 해당 페이지에 포함.

**범위**: 인증 제외, 팀장/팀원 역할 구분 없음 (MVP)

---

## 1. 팀 관리 페이지

**경로**: /team/manage | **좌측 메뉴**: 팀 생성하기, 시작/종료 설정, 태그 설정하기

| # | 기능 | FE | BE | 상세 |
|---|------|:--:|:--:|------|
| T-1 | 팀 생성 | O | O | 팀명 입력 → 저장. 1인당 1팀만 생성 가능. 이미 생성한 팀이 있으면 "이미 지정된 팀이 있습니다" + 팀명만 표시. 필요 시 팀 삭제 버튼으로 팀 삭제 가능(해당 팀에 속한 멤버의 teamId 해제) |
| T-2 | 주간 보고 시작/종료 요일 설정 | O | O | 드롭다운 (0=일~6=토) |
| T-3 | 주간 보고 태그 CRUD | O | O | type=weekly. 추가/수정/삭제 |
| T-4 | 월간(ESM) 태그 CRUD | O | O | type=monthly. 주간 태그의 부모 |

**노출 필드**: team.teamName, team.weekStartDay, team.weekEndDay, tag.tagName, tag.type, tag.parentTagId
**저장 필드**: team(teamName, weekStartDay, weekEndDay), tag(teamId, parentTagId, tagName, type)
**API**: POST /teams, PUT /teams/{teamId}/schedule, POST/PUT/DELETE /teams/{teamId}/tags

## 3. 데일리 슈크럼 작성 페이지

**경로**: / (메인) | **탭**: 데일리 슈크럼 작성 (기본 활성)

| # | 기능 | FE | BE | 상세 |
|---|------|:--:|:--:|------|
| D-1 | 두레이 업무 목록 자동 조회 | O | O | WAS가 Dooray API 대리 호출. 당일 멘션/댓글/담당 업무 |
| D-2 | 데일리 슈크럼 저장 | O | O | report(type=daily) + reportDetail 저장 |
| D-3 | 행 직접 추가 (+) | O | - | 시스템 목록 외 업무 수동 추가 |
| D-4 | 행 제거 (X) | O | O | 저장 전 UI제거, 저장 후 삭제 API |
| D-5 | 주간 보고 태그 선택 (필수) | O | - | 미선택 시 저장 불가. 월간 태그는 parentTagId로 시스템 자동 매핑 |
| D-6 | 데일리 슈크럼 수정 | O | O | |
| D-8 | 동료 협업 기록 저장 | O | O | 팀 동료 이름 선택 → 내용 작성 → 저장 |
| D-9 | 동료 협업 기록 수정/삭제 | O | O | |

**노출 필드**:
- 업무 행: reportDetail.taskTitle(업무 제목), reportDetail.done(한 일), reportDetail.workHours(투입시간), reportDetail.performance(성과), reportDetail.taskLink(업무 링크)
- 태그: reportDetail.tagId → tag.tagName(주간), tag.parentTagId → 월간 태그(자동)
- 동료: member.memberName, peerReport.content

**저장 필드**: report(memberId, staDate, endDate, type=daily), reportDetail(reportId, tagId, taskId, taskTitle, taskLink, done, workHours, performance), peerReport(reportId, peerMemberId, content)

**API**: GET /dooray/tasks, POST /reports/daily, PUT /reports/daily/{reportId}, POST/PUT/DELETE /reports/daily/{reportId}/peer-reports

**입력 규칙**:
- **업무 제목(taskTitle)**: 데일리 슈크럼 작성 화면에서는 **수정 불가**. 링크 입력 시 백엔드가 링크로 제목 조회 후 자동 입력. 저장 후 데일리 보고·자신 작업 노출 시에는 **수정 가능**
- 한 일(done): 필수, 공백/빈문자 불가
- 투입시간(workHours): 소수점 불가, 1시간 단위, 1~24시간(정수)
- 업무 링크(taskLink): 사용자 입력·수정 가능 (URL 입력, 있으면 클릭 시 새 탭)
- 주간 태그: 필수. 미선택 시 저장 불가
- **어필할 성과(performance)**: 선택. **클릭 시 팝업(모달)에서 입력**

## 4. 데일리 보고 페이지

**경로**: /reports/daily | **탭**: 데일리 보고

| # | 기능 | FE | BE | 상세 |
|---|------|:--:|:--:|------|
| R-1 | 데일리 보고 목록 조회 | O | O | 최신순 목록 |
| D-6 | 데일리 보고 수정 | O | O | 수정 시 업무 제목(taskTitle) 포함 수정 가능 |
| D-7 | 데일리 보고 삭제 | O | O | |

**노출 필드**: report.staDate(시작일), report.endDate(종료일) + reportDetail 전 컬럼(taskTitle, done, workHours, performance, taskLink, tagId). **수정 시**: 업무 제목·한 일·투입시간·어필할 성과 등 사용자 수정 가능. **행 클릭/버튼으로 여는 상세 보기 모달에서는 업무 목록은 숨기고, 해당 일자의 어필할 성과(performance)와 동료 협업 기록(peerReport)만 노출.**
**API**: GET /reports/daily?memberId=&from=&to=, PUT /reports/daily/{reportId}, DELETE /reports/daily/{reportId}

## 5. 주간 보고 페이지

**경로**: /reports/weekly | **탭**: 주간 보고

| # | 기능 | FE | BE | 상세 |
|---|------|:--:|:--:|------|
| R-2 | 주간 보고 목록 조회 | O | O | reportTag.workHours(태그별 투입시간 합산) 포함 |
| R-5 | 주간 보고 수정/삭제 | O | O | |
| B-1 | 주간 보고 자동 생성 | - | O | 매일 3AM, 팀 weekStartDay 기준 전주 데이터 |
| - | 주간 보고 수동 생성 | O | O | 수동 생성 버튼 |

**노출 필드**: report.staDate/endDate, reportDetail(taskTitle, done, workHours, tagId), reportTag.workHours(태그별 합산)
**API**: GET /reports/weekly?memberId=&from=&to=, PUT/DELETE /reports/weekly/{reportId}, POST /reports/weekly

**배치 로직** (B-1):
- 매일 새벽 3시 실행
- 오늘이 weekStartDay인 팀의 팀원 대상
- 전주 데일리 보고 → 태그 기준 그룹핑 → 중복 업무 합산(done 개행 연결, workHours 합산) → report + reportDetail + reportTag 생성

## 6. 월간 보고 페이지 (GRM)

**경로**: /reports/monthly | **탭**: 월간 보고

| # | 기능 | FE | BE | 상세 |
|---|------|:--:|:--:|------|
| R-3 | 월간 보고 목록 조회 | O | O | reportTag.workHours + AI 요약 포함 |
| R-6 | 월간 보고 수정/삭제/재생성 | O | O | |
| B-2 | 월간 보고 자동 생성 | - | O | 매월 25일 5AM, 1~25일 구간 |
| - | 월간 보고 수동 생성 | O | O | 수동 생성 버튼 |
| A-1 | AI 요약 생성 | - | O | 태그별 200자 내외 |

**노출 필드**: report.staDate/endDate, reportDetail(taskTitle, done, workHours, tagId), reportTag.workHours(태그별 합산), reportTag.aiSummaryContent(AI 요약)
**API**: GET /reports/monthly?memberId=&from=&to=, PUT/DELETE /reports/monthly/{reportId}, POST /reports/monthly

**배치 로직** (B-2):
- 매월 25일 새벽 5시 실행

**AI 요약** (A-1):
- 입력: done, workHours, performance, taskTitle 태그별 합친 텍스트
- 출력: 200자 내외. 초과 시 잘라내기 또는 재요청

## 7. 리뷰 생성 페이지 (성과 보고)

**경로**: /reports/review | **탭**: 성과 보고

| # | 기능 | FE | BE | 상세 |
|---|------|:--:|:--:|------|
| R-4 | 성과 보고 목록 조회 | O | O | reportTag.workHours + AI 요약 포함 |
| R-7 | 성과 보고 생성 (날짜 지정) | O | O | 시작일/종료일 입력 (기본 1/1~12/31). 기간 겹침 시 오류 |
| R-8 | 성과 보고 수정/삭제 | O | O | |
| A-2 | AI 요약 생성 | - | O | 태그별 500자 내외 |

**노출 필드**: report.staDate/endDate, reportDetail(taskTitle, done, workHours, performance, tagId), reportTag.workHours, reportTag.aiSummaryContent(AI 요약)
**API**: GET /reports/review?memberId=&from=&to=, POST /reports/review, PUT/DELETE /reports/review/{reportId}

**생성 로직** (R-7):
- 사용자가 시작일/종료일 지정 → 해당 구간 데일리 보고 중 performance 있는 업무만 → 태그별 분류·합산 → AI 요약 생성(태그별 500자) → report + reportDetail + reportTag 저장
- 기간 겹침 검증: 기존 성과 보고의 staDate~endDate와 overlap 시 "이미 생성된 범위와 겹칩니다" 에러

**AI 요약** (A-2):
- 입력: performance 있는 업무의 done, workHours, performance, taskTitle 태그별 합친 텍스트
- 출력: 500자 내외
