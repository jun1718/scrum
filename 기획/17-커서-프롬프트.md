## TypeScript 타입 (DDL 기반)

> DDL 원본: `ddl/ddl.sql` — 7개 테이블. 아래 타입은 DDL 컬럼의 camelCase 변환.

// types/index.ts

interface Team {
  teamId: number;
  teamName: string;
  weekStartDay: number | null;  // 0=일~6=토
  weekEndDay: number | null;
  createdAt: string;
  createdMemberId: number;
}

interface Member {
  memberId: number;
  teamId: number | null;
  doorayMemberId: string;
  memberName: string;
  createdAt: string;
  createdMemberId: number;
}

interface Tag {
  tagId: number;
  teamId: number;
  parentTagId: number | null;  // null=monthly, 있으면 weekly
  tagName: string;
  type: 'weekly' | 'monthly';
  createdAt: string;
  createdMemberId: number;
}

interface Report {
  reportId: number;
  memberId: number;
  staDate: string;       // YYYY-MM-DD
  endDate: string;
  type: 'daily' | 'weekly' | 'monthly' | 'review';
  createdAt: string;
  createdMemberId: number;
}

interface ReportDetail {
  reportDetailId: number;
  reportId: number;
  tagId: number;
  taskId: number;
  taskTitle: string;
  taskLink: string;
  done: string;           // 해당 업무로 한 일 (필수)
  workHours: number;      // 투입시간
  performance: string | null;  // 어필할 성과
  createdAt: string;
  createdMemberId: number;
}

interface PeerReport {
  peerReportId: number;
  reportId: number;
  peerMemberId: number;
  content: string | null;
  createdAt: string;
  createdMemberId: number;
}

interface ReportTag {
  reportTagId: number;
  reportId: number;
  tagId: number;
  workHours: number;
  type: 'weekly' | 'monthly' | 'review';
  aiSummaryContent: string | null;  // monthly, review 시 AI 요약
  createdAt: string;
  createdMemberId: number;
}


## Mock 데이터 요구사항

mock JSON 파일을 public/mock/ 에 생성. 현실적인 샘플 데이터:
- 팀 1개 (팀명: "근태개발파트", weekStartDay: 3(수), weekEndDay: 2(화))
- 멤버 3명 (최겸준, 김세미, 최은지)
- 주간 태그 4개 (두레이 개발, 슈크럼 개발, 코드리뷰, 기타), 월간 태그 2개 (두레이, 슈크럼)
- 데일리 보고 5일치 (3/3~3/7)
- 주간 보고 1건, 월간 보고 1건
- 성과 보고 1건
- 동료 협업 기록 2건
- reportTag에 AI 요약 샘플 포함

mock 데이터는 fetch('/mock/reports.json') 방식으로 로드.
나중에 BE 연동 시 fetch URL만 교체하면 되도록 useMockData 훅으로 추상화.

## 주의사항
- **인증/로그인 화면 없음.** 사용자는 이미 로그인된 상태로 가정. currentUser는 mock 데이터에서 고정 (멤버 1번).
- BE 연동 없음. 모든 데이터는 mock JSON + localStorage.
- 저장/수정/삭제는 localStorage로 처리. 새로고침 시 유지.
- 초기 로드 시 localStorage에 데이터 없으면 mock JSON에서 로드하여 세팅.
- 나중에 BE 연동 시 useMockData 훅의 fetch URL만 교체하면 되는 구조로.
- 배치(주간/월간 자동 생성)와 AI 요약은 mock 데이터로 미리 생성된 결과만 보여줌.
