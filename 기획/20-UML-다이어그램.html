1. ERD Entity Relationship

7개 테이블 간의 관계 - tag 계층, report 타입 분류, UNIQUE 제약

erDiagram
    team {
        BIGINT team_id PK
        VARCHAR team_name
        TINYINT week_start_day "0=일 ~ 6=토"
        TINYINT week_end_day
        DATETIME created_at
        BIGINT created_member_id FK
    }

    member {
        BIGINT member_id PK
        BIGINT team_id FK "NULL 허용"
        VARCHAR dooray_member_id "Dooray ID"
        VARCHAR member_name
    }

    tag {
        BIGINT tag_id PK
        BIGINT team_id FK
        BIGINT parent_tag_id FK "NULL이면 monthly"
        VARCHAR tag_name
        VARCHAR type "weekly or monthly"
    }

    report {
        BIGINT report_id PK
        BIGINT member_id FK
        DATE sta_date
        DATE end_date
        VARCHAR type "daily weekly monthly review"
    }

    report_detail {
        BIGINT report_detail_id PK
        BIGINT report_id FK
        BIGINT tag_id FK
        BIGINT task_id "UK report_id task_id"
        VARCHAR task_title
        VARCHAR task_link
        TEXT done
        DECIMAL work_hours
        TEXT performance "NULL 허용"
    }

    peer_report {
        BIGINT peer_report_id PK
        BIGINT report_id FK "daily만"
        BIGINT peer_member_id FK "UK report_id peer_member_id"
        TEXT content
    }

    report_tag {
        BIGINT report_tag_id PK
        BIGINT report_id FK
        BIGINT tag_id FK
        DECIMAL work_hours "태그별 합산"
        VARCHAR type "weekly monthly review"
        TEXT ai_summary_content "monthly review만"
    }

    team ||--o{ member : "1팀 = N멤버"
    team ||--o{ tag : "1팀 = N태그"
    tag ||--o| tag : "월간 - 주간 parent"
    member ||--o{ report : "1멤버 = N보고"
    report ||--o{ report_detail : "1보고 = N상세"
    report ||--o{ peer_report : "1보고 = N동료기록"
    report ||--o{ report_tag : "1보고 = N태그집계"
    tag ||--o{ report_detail : "1태그 = N상세"
    tag ||--o{ report_tag : "1태그 = N집계"
    member ||--o{ peer_report : "동료 참조"

4. 인증 시퀀스 Sequence

Dooray 토큰 -> JWT 발급 -> 커스텀 필터 검증

sequenceDiagram
    actor User as 사용자
    participant FE as Frontend
    participant BE as Backend
    participant Dooray as Dooray API
    participant DB as MySQL

    User->>FE: Dooray API 토큰 입력
    FE->>BE: POST /login (doorayApiToken)

    BE->>Dooray: GET /common/v1/members/me
    Note right of BE: Header: dooray-api token
    Dooray-->>BE: 200 OK (id, name, userCode)

    alt 신규 사용자
        BE->>DB: SELECT member (dooray_member_id)
        DB-->>BE: 없음
        BE->>DB: INSERT member
        DB-->>BE: OK
    else 기존 사용자
        BE->>DB: SELECT member (dooray_member_id)
        DB-->>BE: Member 반환
    end

    BE->>BE: JWT 생성 (memberId, encryptedToken)
    BE-->>FE: 200 OK (jwt, memberInfo)
    FE->>FE: JWT 저장

    Note over FE,BE: 이후 모든 요청

    FE->>BE: Authorization: Bearer jwt
    BE->>BE: JWT 커스텀 필터 검증
    alt 유효
        BE-->>FE: 정상 응답
    else 만료/무효
        BE-->>FE: 401 Unauthorized
    end
5. 데일리 스크럼 작성 시퀀스 Sequence

Dooray 업무 로드 -> 사용자 입력 -> 저장 (report + detail + peer)

sequenceDiagram
    actor User as 사용자
    participant FE as Frontend
    participant BE as Backend
    participant Dooray as Dooray API
    participant DB as MySQL

    User->>FE: 데일리 스크럼 탭 클릭

    FE->>BE: GET /dooray/tasks
    BE->>BE: JWT에서 doorayToken 복호화
    BE->>Dooray: GET /project/v1/tasks (멘션/댓글/담당)
    Dooray-->>BE: 업무 목록
    BE-->>FE: 가공된 업무 목록

    FE->>FE: 업무 목록 테이블 렌더링

    loop 각 업무 행 입력
        User->>FE: done 입력 [필수]
        User->>FE: workHours 입력
        User->>FE: performance 입력 [선택]
        User->>FE: 주간 태그 선택 [필수]
        FE->>FE: 월간 태그 자동 매핑
    end

    opt 수동 행 추가
        User->>FE: + 버튼 클릭
        FE->>FE: 빈 행 추가
        User->>FE: taskTitle, taskLink 입력
    end

    opt 동료 협업 기록
        User->>FE: 동료 협업 버튼
        User->>FE: 팀원 선택 + 내용 입력
    end

    User->>FE: 저장 버튼 클릭

    FE->>BE: POST /reports/daily
    Note right of FE: staDate, endDate, reportDetails[], peerReports[]

    BE->>DB: INSERT report (type=daily)
    BE->>DB: INSERT report_detail x N
    Note right of BE: UNIQUE check: reportId + taskId
    opt 동료 기록 있으면
        BE->>DB: INSERT peer_report x N
        Note right of BE: UNIQUE check: reportId + peerMemberId
    end
    DB-->>BE: OK

    BE-->>FE: 201 Created
    FE-->>User: 저장 완료
6-1. 주간 보고 자동 생성 - 배치 Sequence

매일 3AM - weekStartDay 해당 팀만, 팀원 단위 개별 트랜잭션

sequenceDiagram
    participant Scheduler as Spring Scheduler (매일 3AM)
    participant Batch as BatchService
    participant DB as MySQL

    Scheduler->>Batch: triggerWeeklyGeneration()

    Batch->>DB: SELECT team WHERE weekStartDay = 오늘 요일
    DB-->>Batch: 대상 팀 목록

    loop 각 팀
        Batch->>DB: SELECT member WHERE teamId
        DB-->>Batch: 팀원 목록

        loop 각 팀원 (개별 트랜잭션)
            Batch->>DB: SELECT report + detail (daily, 지난주간)
            DB-->>Batch: 일간 보고 상세

            Batch->>Batch: 태그별 그룹핑
            Batch->>Batch: done 합치기 (줄바꿈)
            Batch->>Batch: workHours 합산
            Batch->>Batch: 중복 업무 제거

            Batch->>DB: INSERT report (type=weekly)
            Batch->>DB: INSERT report_detail x N
            Batch->>DB: INSERT report_tag x N
            DB-->>Batch: OK
        end
    end

    Note over Scheduler,DB: 팀원 단위 개별 트랜잭션 - 1명 실패해도 나머지 정상
6-2. 월간 보고 자동 생성 - 배치 Sequence

매월 25일 5AM - 1일~25일 집계 + AI 요약 200자

sequenceDiagram
    participant Scheduler as Spring Scheduler (25일 5AM)
    participant Batch as BatchService
    participant AI as AISummaryService
    participant Claude as Claude API
    participant DB as MySQL

        end

        DB-->>Batch: OK
    end

10. 팀 공유 시퀀스 Sequence

팀원 보고서 조회 시 민감 정보(performance, peer) 필터링 + 가동률 산출

sequenceDiagram
    actor User as 팀원
    participant FE as Frontend (팀 공유)
    participant BE as Backend
    participant DB as MySQL

    User->>FE: 팀 공유 탭 클릭 (일간/주간/월간)
    FE->>BE: GET /teams/{teamId}/reports/{type}?date=

    BE->>BE: JWT에서 memberId, teamId 추출

    BE->>DB: SELECT report + detail (팀, 타입, 날짜)
    DB-->>BE: 팀원 보고 목록

    BE->>BE: 민감 정보 필터링
    Note right of BE: performance 제외, peer_report 제외, done/workHours/tag 포함

    BE->>BE: 가동률 산출
    Note right of BE: utilization = workHours / 일총시간 x 100

    BE-->>FE: 필터링된 보고 + 가동률(%)
