# 8. 사용자 행동별 기능 정의

**노출 구간 DB 컬럼·한글명**: 각 절 "(노출 데이터)"에 **DB 컬럼명(한글명)** 표기. 전체 매핑은 **13-ERD-논리검토-및-컬럼한글매핑.md** 참고.

## 팀장 — 최초 슈크럼 웹 오픈 후 팀 세팅

1. **팀 생성하기**: 메인 > 좌측 메뉴 > 팀 생성하기 > 팀명 입력 > 저장. **(저장)** team.teamName(팀명). **1인당 1팀만 생성 가능.** 이미 생성한 팀이 있으면 폼 대신 "이미 지정된 팀이 있습니다" + 팀명만 표시.
2. **팀 시작/종료일자 설정하기**: 주간보고 **시작요일·종료요일** 설정. **(저장)** team.weekStartDay, team.weekEndDay.
3. **태그 설정하기**: 주간보고 태그 먼저, 그다음 월간(ESM) 그룹핑 태그. **(저장)** tag.teamId, tag.parentTagId, tag.tagName, tag.type.

## 팀원 — 최초 오픈 후 메인 페이지 접속

1. 좌측 메뉴 있고, 중앙 **"팀을 지정해주세요"**. 팀 없으면 "오늘 한 일" 영역 미표시.
2. **팀 지정하기** 클릭 > 팝업에서 팀 이름 입력 또는 목록 선택 (자동 검색). **(노출)** team.teamId(팀 ID), team.teamName(팀명). **BE (한 트랜잭션)**: Member 데이터 생성 후 member.teamId 갱신. 두레이 API로 Member 정보 조회해 이름 등 자동 저장. **(저장)** member.memberId, member.teamId(소속 팀 ID), member.doorayMemberId, member.memberName(회원명). **이미 Member 존재 시**: "현재 스프린톤에서는 다중 팀 미지원. 추후 개선 예정" 에러.
3. 완료 후 새로고침 → 공통 행동으로 이어짐.

## 사용자 공통 — 팀 지정 후 메인 (데일리 슈크럼 작성)

1. 당일 "내가 멘션된 업무", "내가 쓴 댓글", "등록자/담당자/참조자 수정 업무" 목록 노출. **(노출·저장)** reportDetail.taskTitle(업무 제목), reportDetail.done(해당 업무로 한 일), reportDetail.workHours(투입시간), reportDetail.performance(어필할 성과), reportDetail.taskLink(업무 링크). 주간 보고 태그: reportDetail.tagId → tag.tagName(type=weekly). 월간: tag.parentTagId로 유도. **+** 버튼(직접 추가), **X** 버튼(행 제거).
   - **컬럼 규칙**: **업무 제목**: 데일리 슈크럼 **작성** 화면에서는 **사용자 수정 불가**. 링크만 입력하면 백엔드가 해당 링크의 제목을 조회해 넣어줌. **저장 후** 데일리 보고·자신이 작업한 내용 노출 시에는 **수정 가능**. 해당 업무로 한 일: **필수**(공백/빈문자 불가, 최소 1글자). **어필할 성과**: 선택. **클릭 시 팝업(모달)에서 입력**. 작성 화면에서 사용자 입력/수정 가능: **업무 링크**, 한 일, 투입시간. **투입시간(workHours): 소수점 불가, 1시간 단위, 1~24시간 범위(정수).** **투입률: 숫자만.**
2. **동료 협업 기록**: 버튼 클릭 > 팀 소속 동료 이름 노출 > 이름 선택 > 협업 내용 작성 > 저장. 여러 동료 저장/수정/삭제 가능. **(노출)** member.memberName(회원명). **(저장)** peerReport.reportId(데일리 보고 ID), peerReport.peerMemberId(동료 회원 ID), peerReport.content(동료 기록 내용).
3. **저장** 버튼. 저장 후에도 행 수정·삭제 가능.

## 사용자 공통 — 상단 탭 (대시보드 UI)

| 한글(주) | 영어(괄호) | 출처 |
|----------|------------|------|
| 데일리 슈크럼 작성 | (Daily Scrum Write) | 작성 화면. 저장 시 report.type=daily |
| 데일리 보고 | (Daily Report) | report.type=daily |
| 주간 보고 | (Weekly Report) | report.type=weekly |
| 월간 보고 | (Monthly Report / GRM) | report.type=monthly |
| 성과 보고 | (Review Report) | report.type=review |

- **데일리 보고** 탭 클릭: 자신이 작성한 데일리 슈크럼 **최신순** 목록. **(노출)** report.reportId, report.staDate(보고 시작일), report.endDate(보고 종료일) + reportDetail 전 컬럼(taskTitle, done, workHours, performance, taskLink, tagId 등). 수정/삭제/생성 가능. **행을 클릭하거나 상세 보기 버튼을 누르면 해당 일자의 상세 모달이 열리고, 이 모달에는 업무 목록 대신 어필할 성과(performance)와 동료 협업 기록(peerReport)만 노출.**
- **주간 보고** 탭 클릭: **(노출)** report(type=weekly), report.staDate/endDate, reportDetail, 태그별 workHours(조회 시 SUM 계산). 수정/삭제/생성 가능. **화면 상단에 \"주간 보고 생성\" 수동 생성 버튼 노출.**
- **월간 보고** 탭 클릭: **(노출)** report(type=monthly), reportDetail, 태그별 workHours(조회 시 SUM 계산), reportDetail.aiSummary(AI 요약 내용). 수정/삭제/재생성 가능. **화면 상단에 \"월간 보고 생성\" 수동 생성 버튼 노출.**
- **성과 보고** 탭 클릭: **(노출)** `report_detail_tag`만 표시 — 월간태그명(tag.tagName) | 투입시간(report_detail_tag.work_hours) | 투입률(%) | AI 요약(report_detail_tag.ai_summary). 날짜 지정 후 생성 버튼으로 생성. **화면 상단에 \"성과 생성\" 버튼 노출.**
  - **상세보기 버튼**: 태그별 해당 기간 reportDetail 조회 API 호출 (`GET /api/reports/review/{reportId}/details?tagId={tagId}`). 반환: 업무명(업무링크 하이퍼링크) | 한 일(done) | 업무별 AI 요약(reportDetail.aiSummary) | 어필할 성과(reportDetail.performance). 태그 내 reportDetail.aiSummary + reportDetail.performance 쌍을 합쳐서 태그별 AI 요약이 도출되었음을 UI에 명시 (사용자 검증용).
- **모든 보고 공통**: 원래 수정 불가로 막으려 했으나, 상위 레벨 데이터 재생성 시 반영을 위해 **수정 허용**.

## 사용자 공통 — 팀 업무 공유 메뉴

- **데일리/주간/월간 보고** 탭: 해당 팀 소속 팀원들의 일자별 보고 목록 (투입률 %, 총 투입시간). **투입률(%)** = 특정 업무 workHours / 당일 전체 workHours 합계 × 100. **(노출)** report.reportId, report.memberId → member.memberName(회원명), report.staDate/endDate, reportDetail.taskTitle(업무 제목), reportDetail.workHours(투입시간), 태그별 workHours(조회 시 SUM 계산). **노출 제외(민감)** reportDetail.performance(어필할 성과), peerReport.content(동료 기록).
