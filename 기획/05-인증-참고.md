# 5. 인증 참고

## 로그인 흐름

1. 사용자 최초 메인 페이지 접근 시 **DOORAY_API_TOKEN** 입력 (로그인 개념).
2. FE → 슈크럼 WAS **POST /login** 호출.
3. WAS가 두레이 토큰으로 **Dooray Members Open API** 호출 (`GET https://api.dooray.com/common/v1/members/me`, `Authorization: dooray-api ${DOORAY_API_TOKEN}`).
4. 반환값의 `id`로 슈크럼 DB **Member** 테이블 검색.
   - 존재하면 → JWT 발급 후 FE 전달.
   - 없으면 → Member 데이터 생성 후 JWT 발급 후 FE 전달.
5. FE는 이후 요청마다 **JWT** 전달. WAS는 JWT 검증 후 claim 복호화해 사용.

## JWT claim

- **memberId**
- **dooray_api_token** (암호화하여 저장)

## 참고

- **Spring Security 미사용.**
- Backend DB에는 사용자 DOORAY_API_TOKEN을 저장하지 않음. FE에서 로컬 저장 후 API 요청 시마다 JWT로 전달.
