# 18. 디자인 가이드

Dooray 업무 화면 레이아웃을 기반으로 한 슈크럼 UI 디자인 가이드.

---

## 전체 레이아웃 (3단 구조)

```
┌──────┬─────────────────┬──────────────────────────────────────────┐
│ 아이콘│   좌측 패널       │          메인 콘텐츠 영역                  │
│  바  │                 │                                          │
│ 48px │    240px        │           나머지 전체                      │
│      │                 │                                          │
│ 📋   │ 팀 생성하기      │  ┌─────────────────────────────────────┐  │
│ 📊   │ 시작/종료 설정    │  │ 상단 탭 바                           │  │
│ 👥   │ 태그 설정하기     │  │ 데일리 작성 | 데일리 | 주간 | 월간 | 성과│  │
│ ⚙️   │ 팀 지정하기      │  ├─────────────────────────────────────┤  │
│      │ ─────────────   │  │                                     │  │
│      │ 내 업무보고(메인)  │  │         페이지 콘텐츠                 │  │
│      │ 팀 업무 공유      │  │                                     │  │
│      │                 │  │                                     │  │
│      │                 │  └─────────────────────────────────────┘  │
└──────┴─────────────────┴──────────────────────────────────────────┘
```

### 1. 최좌측 아이콘 바 (IconSidebar)

Dooray 최좌측과 동일한 세로 아이콘 메뉴. "슈크럼" 서비스가 여기에 추가된다고 가정.

```
스타일:
- 너비: 48px, 고정
- 배경: bg-[#1B1D21] (Dooray와 유사한 다크 차콜)
- 높이: 100vh
- 아이콘: 24x24, 세로 중앙 정렬, 간격 8px
- 활성 아이콘: 좌측 3px 흰색 보더 + 아이콘 밝게
- hover: bg-white/10 rounded

아이콘 목록 (위→아래):
- 슈크럼 로고 (활성 상태, 항상 하이라이트)
- 구분선
- (하단) 사용자 아바타 (원형, 32x32)

MVP에서는 아이콘 바를 장식용으로만 렌더링. 클릭 동작 없음.
슈크럼 아이콘만 활성 상태로 표시.
```

### 2. 좌측 네비게이션 패널 (NavPanel)

Dooray 좌측 프로젝트 트리와 유사한 메뉴 패널.

```
스타일:
- 너비: 240px, 고정
- 배경: bg-white
- 우측 보더: border-r border-gray-200
- 높이: 100vh

상단 영역:
- 서비스명 "슈크럼" (font-bold, text-lg, px-4 py-3)
- 팀명 표시 (text-sm text-gray-500, 팀 지정 시)

메뉴 항목 스타일:
- px-4 py-2.5
- text-sm text-gray-700
- hover: bg-gray-100 rounded-md (mx-2)
- 활성: bg-blue-50 text-blue-600 font-medium (mx-2, rounded-md)
- 아이콘(16x16) + 텍스트, gap-3

메뉴 구조:
─── 팀 설정 (섹션 헤더, text-xs text-gray-400 uppercase, px-4 pt-4 pb-1)
    팀 생성하기
    시작/종료일자 설정
    태그 설정하기
    팀 지정하기
─── 보고 (섹션 헤더)
    내 업무보고 확인   ← 기본 선택
    팀 업무 공유

구분선: border-t border-gray-100, my-2
```

### 3. 메인 콘텐츠 영역 (MainContent)

Dooray 우측 콘텐츠 영역과 동일한 구조.

```
스타일:
- flex-1 (나머지 공간 전체)
- 배경: bg-gray-50
- overflow-y: auto

내부 구조:
┌─────────────────────────────────────────┐
│ 상단 헤더 (h-14, bg-white, border-b)     │
│ - 페이지 제목 (font-semibold)             │
│ - 우측: 날짜 표시 또는 액션 버튼            │
├─────────────────────────────────────────┤
│ 탭 바 (bg-white, border-b)              │
│ - 탭 항목: px-4 py-3, text-sm           │
│ - 활성 탭: border-b-2 border-blue-500    │
│            text-blue-600 font-medium    │
│ - 비활성: text-gray-500 hover:text-700   │
├─────────────────────────────────────────┤
│ 콘텐츠 (p-6)                             │
│ - 카드: bg-white rounded-lg shadow-sm    │
│         border border-gray-200 p-4      │
│ - 테이블: 카드 안에 배치                   │
└─────────────────────────────────────────┘
```

## 컴포넌트별 디자인

### 테이블 (ReportTable)

Dooray 업무 목록 스타일.

```
- 헤더: bg-gray-50, text-xs text-gray-500 uppercase, font-medium
- 행: border-b border-gray-100, hover:bg-gray-50
- 셀: px-4 py-3, text-sm
- 스트라이프 없음 (Dooray 스타일은 hover 강조)
- 행 클릭 시 상세 펼침 또는 모달
```

### 데일리 슈크럼 입력 행 (ScrumRow)

```
- 카드 형태: bg-white rounded-lg border border-gray-200 p-4 mb-3
- 행 내부: grid 또는 flex, gap-4
- 필드 라벨: text-xs text-gray-500, mb-1
- 입력 필드: border border-gray-300 rounded-md px-3 py-2 text-sm
             focus:ring-2 focus:ring-blue-500 focus:border-blue-500
- 필수 필드: 라벨 옆 빨간 * (text-red-500)
- 태그 선택: 드롭다운, 선택 시 태그 칩(bg-blue-100 text-blue-700 rounded-full px-2 py-0.5 text-xs)
- + 버튼: border-dashed border-2 border-gray-300, 전체 너비, py-3, text-gray-400
           hover:border-blue-400 hover:text-blue-500
- X 버튼: 행 우상단, text-gray-400 hover:text-red-500

행 레이아웃:
┌──────────────────────────────────────────────────────┐
│ [업무 제목 ─────────────────] [업무 링크 🔗]    [X]   │
│ [한 일 (필수) ──────────────────────────────────────] │
│ [투입시간 ──] [태그 선택 ▼ ──] [월간태그: 자동] [성과 ─]│
└──────────────────────────────────────────────────────┘
```

### 버튼

```
Primary:   bg-blue-500 hover:bg-blue-600 text-white px-4 py-2 rounded-md text-sm font-medium
Secondary: bg-white border border-gray-300 hover:bg-gray-50 text-gray-700 px-4 py-2 rounded-md text-sm
Danger:    bg-white border border-gray-300 hover:bg-red-50 text-red-600 px-4 py-2 rounded-md text-sm
Ghost:     bg-transparent hover:bg-gray-100 text-gray-600 px-3 py-2 rounded-md text-sm
```

### 모달 (PeerReportModal, ReviewCreateModal)

```
- 오버레이: bg-black/50, fixed inset-0
- 모달 박스: bg-white rounded-xl shadow-xl, max-w-lg, mx-auto, mt-20
- 헤더: px-6 py-4 border-b, font-semibold text-lg
- 본문: px-6 py-4
- 푸터: px-6 py-4 border-t, flex justify-end gap-3
```

### 태그 칩

```
주간 태그: bg-blue-100 text-blue-700 rounded-full px-2.5 py-0.5 text-xs font-medium
월간 태그: bg-purple-100 text-purple-700 rounded-full px-2.5 py-0.5 text-xs font-medium
```

### 빈 상태 (Empty State)

```
- 팀 미지정: 중앙 정렬, 아이콘(48x48 gray-300) + "팀을 지정해주세요" (text-gray-500 text-lg)
             + "팀 지정하기" 버튼 (Primary)
- 데이터 없음: 중앙 정렬, "아직 보고가 없습니다" (text-gray-400)
```
