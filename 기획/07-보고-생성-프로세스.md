# 7. 보고 생성 프로세스

**용어**: 데일리 보고 = report(type='daily') 1건 + reportDetail 목록. 주간/월간/성과 보고는 각각 type weekly, monthly, review. (06 참고.)

## 보고 생성 공통 프로세스

1. 당일 배치 실행 시 "오늘이 보고 **시작일**인 팀"의 팀원 목록 조회 → 타겟별 반복.
2. **주간**: 매일 **새벽 3시**. 해당 날짜 기준 **전주** 주간 보고 생성.
3. **월간**: **매월 25일 새벽 5시**. **당월** 1일~25일 구간 월간 보고 생성.
4. **성과 보고**: 배치 없음. 사용자 직접 생성.

대상 타겟들의 시작일~종료일 **데일리 보고** 조회 (MyBatis list 조회 후 Java에서 map 세팅: memberIdToReportMap. report 안에 **reportDetail** 포함 — `<collection>` 등으로 한 번에 조회) → 타겟별로 주간/월간/성과 보고 생성.

## 타겟별 보고 생성 (주/월/성과 공통)

- **하나의 트랜잭션**(멤버별 개인 트랜잭션).

### A. 업무 중복 제거 및 report 저장

1. 멤버에 해당하는 해당 주/월 보고 목록 조회 (memberIdToReportListMap 사용).
2. **report** 및 **reportDetail** 생성(업무 합산 내용 넣을 그릇).
3. 1번 결과를 **태그 기준**으로 업무 그룹핑(중복 업무 하나로 합침). done, workHours, performance 합침 — 문자열은 개행 2번 후 연결, workHours는 합산.
4. 최종 **report**, **reportDetail** DB 저장.

### B. 보고 태그별 반정규화 데이터 삽입

1. A 결과물을 **태그별**로 분류(주간/월간에 맞게).
2. 태그별 **workHours** 합산 후 **reportTag**에 저장. (AI 요약은 이 단계에서 하지 않음. 사용자 "생성하기" 버튼 시 처리.)
3. **reportTag** 테이블 저장.

## AI 요약 생성

- **월간 보고**: reportDetail 태그별 분류·합침 → AI 요약 (200자 내외) → aiSummaryContent 저장.
- **성과 보고**: performance 있는 업무만 조회 → 태그별 분류·합침 → AI 요약 (500자 내외) → aiSummaryContent 저장.

## AI 요약 규격

- **입력**: done, workHours, performance, taskTitle 태그별 합친 텍스트.
- **출력**: 월간 200자 내외, 성과 500자 내외. 초과 시 팀 정책(잘라내기 또는 재요청).
