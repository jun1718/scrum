# 7. 보고 생성 프로세스

**용어**: 데일리 보고 = report(type='daily') 1건 + reportDetail 목록. 주간/월간/성과 보고는 각각 type weekly, monthly, review. (06 참고.)

## 사전 조건

- `task_id`는 데일리 슈크럼 저장 시 `task_link`에서 추출하여 저장한다.

## 보고 생성 공통 프로세스 — 동일 업무 합치기 (공통 메서드)

주간/월간/성과 보고 모두 동일한 합치기 로직을 사용한다 (`task_id` 기준).

1. 하위 보고의 `report_detail` 목록 조회.
2. **동일 업무 합치기** (`task_id` 기준):
   - `work_hours`: 모두 합산.
   - `done`: `\n\n` (개행 2번) 기준 합치기.
   - `performance`: `\n\n` 기준 합치기.
   - `ai_summary`: `\n\n` 기준 합치기 (월간/성과만 해당, 주간은 원래 없음).
3. 최신일자 하위 보고의 동일 업무에서 `task_title`, `task_link`, `tag_id` 사용.
4. 최신일자 `tag_id` 기준으로 **태그별 정렬**.
5. 새 `report` + `report_detail` 레코드 생성 (하위 보고와 의존성 없음, 태그별 정렬 순서 유지).

## 주간보고 생성

- **소스**: 일간보고의 `report_detail`.
- **날짜 범위**: `team.weekStartDay`, `team.weekEndDay` 기준 자동 산출.
- **트리거**: 사용자 수동 생성 버튼 또는 배치 (매일 새벽 3시, 오늘이 주간보고 시작일인 팀).

1. 공통 합치기 프로세스 수행 (소스: 일간보고).
2. **AI 요약** (업무별, **모든 업무** 대상 — 주간보고는 기본 aiSummary가 없으므로 전체 수행):
   - Dooray API로 업무 제목+본문 조회 + `done` → Claude API 개조식 요약 → `report_detail.ai_summary` 저장.
   - **Dooray API 업무 조회는 주간보고 생성 시에만 수행.**

## 월간보고 생성

- **소스**: **주간보고**의 `report_detail`. (주간보고와 동일한 공통 메서드로 처리)
- **날짜 범위**: 사용자가 **월만 지정** → 해당월 1일~말일.
- **트리거**: 사용자 수동 생성 버튼.

1. 공통 합치기 프로세스 수행 (소스: 주간보고. `ai_summary`도 합침 대상).
2. **AI 요약** (업무별, **동일 업무(중복)가 있었던 업무만** 대상):
   - 합쳐진 aiSummary를 context로 Claude API 개조식 요약 → `report_detail.ai_summary` 저장.
   - 중복 없는 업무는 주간보고의 기존 aiSummary 그대로 사용.

## 성과보고 생성

- **소스**: **월간보고**의 `report_detail` (performance 필터 없이 **모든 업무** 포함).
- **날짜 범위**: 사용자가 **시작일/종료일 지정** (기본 1/1~12/31). 기존 성과보고와 기간 중복 시 409 오류.
- **트리거**: 사용자 직접 생성 (배치 없음).

1. 공통 합치기 프로세스 수행 (소스: 월간보고. `ai_summary`도 합침 대상).
2. **업무별 AI 요약** (**동일 업무(중복)가 있었던 업무만** 대상):
   - 합쳐진 aiSummary를 context로 Claude API 개조식 요약 → `report_detail.ai_summary` 저장.
   - 중복 없는 업무는 월간보고의 기존 aiSummary 그대로 사용.
3. **태그별 공수 합산** (성과보고만): 태그 내 모든 `report_detail.work_hours` 합산 → `report_detail_tag.work_hours` 저장.
4. **태그별 AI 요약** (필수, 모든 업무 대상): 태그당 1회 Claude API 호출 (context 희석 방지).
   - 프롬프트 파일 별도 참조.
   - context JSON:
     ```json
     {
       "tagId": 123,
       "tagName": "프로젝트A",
       "tasks": [
         {
           "aiSummary": "업무 본문을 가져와서 본문/제목/해당 업무에 대해 한일들을 context로 넘겨서 요약한 값입니다.",
           "performance": "사용자가 해당 업무에 대해 어필하고 싶어하는 내용으로 등록한 값입니다."
         }
       ]
     }
     ```
   - 반환 JSON: `{ "tagId": 123, "tagName": "프로젝트A", "aiSummary": "요약 결과..." }`
   - 결과를 `report_detail_tag.ai_summary`에 저장 (500자 내외).

## 구현 시 공통화 포인트

- **동일 업무 합치기 메서드**: 주간/월간/성과 모두 동일 로직.
- **업무별 AI 요약 메서드**: input만 다르고 요약 호출/저장 로직은 공통.
  - 주간: 모든 업무 대상. Dooray API 조회는 주간보고에서만.
  - 월간/성과: 중복 있었던 업무만 대상.
- **태그별 AI 요약**: 성과보고 전용 로직 (공통화 대상 아님).
